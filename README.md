## Что работает

- __Поиск пользователей GitHub__: возможность найти профиль любого пользователя по его юзернейму через поисковую строку.

- __Отображение основной информации__: вывод аватара, имени, биографии и статистики (количество подписчиков, подписок и публичных репозиториев).

- __Список последних репозиториев__: показ 5 последних созданных репозиториев с их названиями, описаниями и количеством звезд.

- __Анализ языков программирования__: формирование круговой диаграммы и легенды с процентным соотношением используемых языков на основе всех репозиториев пользователя.

- __История поиска__: возможность добавлять нескольких пользователей на одну страницу.

- __Управление списком__: функции удаления последнего добавленного пользователя или полного сброса страницы.

- __Уведомления__: система всплывающих сообщений об успешном поиске или возникших ошибках.

- __Адаптивный и анимированный интерфейс__: современный дизайн с адаптивкой и плавными анимациями появления карточек.

## Что я узнал во время работы

1) Стандарт документирования и аннотирования JSdoc, где можно указывать какие параметры ожидает функция с определенным типом, что она возвращает и какую ошибку выдает
```
/**
 * Получает информацию о пользователе GitHub по его имени.
 * @param {string} username - Юзернейм на GitHub
 * @returns {Promise<Object>} Данные пользователя
 * @throws {Error} Ошибка при получении данных
 */
```  
2) Трюк для избежания rate limits
```
for (let i = 0; i < repos.length; i += 5) {
            const batch = repos.slice(i, i + 5);

            const results = await Promise.allSettled(
                batch.map(repo =>
                    fetch(repo.languages_url).then(res => {
                        if (!res.ok) throw new Error(`Ошибка получения языков для ${repo.name}`);
                        return res.json();
                    })
                )
            );

            for (const result of results) {
                if (result.status === 'fulfilled') {
                    const repoLanguages = result.value;
                    for (const [lang, bytes] of Object.entries(repoLanguages)) {
                        languageMap[lang] = (languageMap[lang] || 0) + bytes;
                    }
                }
                // Если запрос не удался — просто пропускаем этот репозиторий
            }
        }
```
Берем каждый пятый элемент и создаем батч-массив с отрезком из этих пяти элементов, запускаем 5 парралельных запроса по циклу map, получая json-ответы и формируя из них массив languageMap.  
3) `await Promise.allSettled()` получает массив данных, обрабатывает каждый его элемент независимо от результата и ошибок и возвращает массивы со статусами и результатами  
4) `entries.sort((a, b) => b - a);` - метод сортировки. Если в результате b - a получится положительное число, значит, b больше и его нужно поставить впереди. Таким образом реализуется сортировка по убыванию  
5) `top.map(([name, bytes]) => ({ name, bytes }))` - перевод массива в объект  
6) `chartData.reduce((sum, item) => sum + item.bytes, 0);` - проходится по циклу с переменной, которая используется для сбора данных. Имеется начальное значение 0  
7) `toFixed(1)` - округляет до n-го количества знаков после запятой  
8) Двойной рендер:
```
requestAnimationFrame(() => {
        requestAnimationFrame(() => {
            const segments = card.querySelectorAll('.lang-segment');
            segments.forEach(seg => {
                seg.style.strokeDashoffset = seg.dataset.finalOffset;
            });
        });
    });
```
При первом вызове функции браузер отрисовывает элементы, запуская остальные функции, поэтому есть риск, что браузер не отрисует анимацию для оптимизации. При втором вызове функции браузер уже должен будет все отрисовать и он сможет исправно заменить значение strokeDashoffset и применить анимацию
